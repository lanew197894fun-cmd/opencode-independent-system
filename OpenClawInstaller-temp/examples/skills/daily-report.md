# 每日報告技能

## 描述
產生每日工作報告和待辦事項總結，幫助你回顧一天的工作成果。

## 觸發條件
- 使用者說 "產生日報" 或 "每日報告" 或 "今日總結"
- 每天晚上 20:00 自動觸發（可在配置中調整）

## 執行步驟
1. 收集今日完成的任務和對話摘要
2. 列出未完成的待辦事項
3. 總結今日的重要資訊和決策
4. 產生明日工作計畫建議
5. 統計今日 API 使用量（可選）

## 輸出格式

```
📅 {日期} 每日報告

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ 今日完成
{完成的任務列表，帶時間戳}

📋 待辦事項
{未完成的任務，按優先級排序}

💡 重要資訊
{今日的關鍵資訊和決策}

📊 統計資料
• 對話次數: {count}
• 處理任務: {count}
• API 消耗: {tokens} tokens

🎯 明日建議
{基於今日情況的明日計畫建議}

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 範例

**輸入**: "產生今天的日報"

**輸出**:
```
📅 2026-01-29 每日報告

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ 今日完成
• 09:30 - 完成專案程式碼審查
• 11:00 - 回覆了3封重要郵件
• 14:00 - 完成週報撰寫
• 16:30 - 參加產品評審會議

📋 待辦事項
1. 🔴 [緊急] 修復登入頁面 Bug
2. 🟡 [重要] 準備週五演示 PPT
3. 🟢 [普通] 整理專案文檔

💡 重要資訊

• 新功能測試回饋良好

• 處理任務: 8

🎯 明日建議
• 優先處理登入頁面 Bug
• 預留2小時準備 PPT
• 下午可安排文檔整理工作

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 配置選項

```yaml
skills:
  daily-report:
    enabled: true
    schedule: "0 20 * * *"  # 每天 20:00 觸發
    include_stats: true      # 包含統計資料
    include_suggestions: true # 包含明日建議
    channels:                # 推送渠道
      - telegram
      - discord
```
