# 🔷 飛書機器人配置指南

本文檔詳細介紹如何配置 OpenClaw 的飛書訊息渠道。

## 📋 前置要求

- 已安裝 OpenClaw（執行 `install.sh` 完成安裝）
- 飛書帳號（個人帳號即可，無需企業認證）

## ✨ 特性說明

OpenClaw 飛書渠道具有以下特性：

- ✅ **無需公網伺服器** - 使用 WebSocket 長連線模式
- ✅ **無需企業認證** - 個人帳號即可建立"企業自建應用"
- ✅ **支援私聊和群聊** - 群聊中 @機器人 觸發回覆
- ✅ **支援多媒體訊息** - 圖片、檔案等

## 🚀 配置步驟

### 第一步：建立飛書應用

1. 存取 [飛書開放平台](https://open.feishu.cn/)
2. 登入後點擊右上角「建立應用」
3. 選擇「企業自建應用」（個人帳號也可以建立）
4. 填寫應用名稱（如 "OpenClaw"）和描述

### 第二步：新增機器人能力

1. 進入應用詳情頁
2. 點擊左側選單「新增應用能力」
3. 找到「機器人」能力，點擊新增
4. 確認「機器人」開關已開啟

> 💡 **路徑**: 開發者後台 → 應用詳情 → 新增應用能力 → 機器人

### 第三步：取得應用憑證

在應用詳情頁的「憑證與基礎資訊」中，複製以下資訊：

- **App ID** - 應用唯一識別
- **App Secret** - 應用金鑰（點擊顯示後複製）

⚠️ 請妥善保管 App Secret，不要洩露給他人。

### 第四步：配置權限

1. 點擊左側選單「權限管理」
2. 搜尋並新增以下權限：

| 權限名稱 | 權限識別 | 說明 |
|---------|---------|------|
| 取得與傳送單聊、群組訊息 | `im:message` | 收發訊息（必須） |
| 以應用的身份傳訊息 | `im:message:send_as_bot` | 傳送訊息（必須） |
| 取得群組資訊 | `im:chat:readonly` | 讀取群資訊（推薦） |

### 第五步：發布應用

1. 點擊左側選單「版本管理與發布」
2. 點擊「建立版本」
3. 填寫版本號和更新說明
4. 設定可用範圍（選擇可使用此應用的人/部門）
5. 點擊「儲存」然後「申請發布」

> 💡 內部應用通常會自動審核通過，無需等待。

### 第六步：在 OpenClaw 中配置飛書

執行配置選單：

```bash
bash ~/.openclaw/config-menu.sh
```

1. 選擇 `[3] 訊息渠道配置`
2. 選擇 `[7] 飛書 (Feishu)`
3. 按提示輸入 **App ID** 和 **App Secret**
4. 配置完成後選擇「是」重啟 Gateway

### 第七步：配置事件訂閱（長連線）

> ⚠️ **重要**: 此步驟需要 OpenClaw 服務已啟動，否則無法儲存長連線設定。

1. 回到飛書開放平台，進入應用詳情
2. 點擊左側選單「事件與回調」
3. 選擇「**使用長連線接收事件**」（不是 Webhook）
4. 點擊「新增事件」，新增以下事件：

| 事件名稱 | 事件識別 | 說明 |
|---------|---------|------|
| 接收訊息 | `im.message.receive_v1` | 必須 |
| 訊息已讀 | `im.message.message_read_v1` | 可選 |
| 機器人進群 | `im.chat.member.bot.added_v1` | 可選 |

5. 點擊「儲存」

> 💡 使用長連線模式**無需填寫 Webhook 位址**，無需公網伺服器。

### 第八步：新增機器人到群組

**方法一：在飛書客戶端新增**

1. 打開飛書，進入目標群組
2. 點擊右上角群設定（⚙️ 圖示）
3. 點擊「群機器人」→「新增機器人」
4. 搜尋你的機器人名稱並新增

**方法二：建立新群並新增**

1. 建立一個新的群組
2. 在建立時直接新增機器人
3. 或建立後在群設定中新增

## 🧪 測試配置

### 方法一：使用配置選單測試

```bash
bash ~/.openclaw/config-menu.sh
```

1. 選擇 `[7] 快速測試`
2. 選擇 `[5] 測試飛書機器人`
3. 系統會自動讀取已配置的憑證
4. 可選輸入群組 Chat ID 傳送測試訊息

### 方法二：直接在群裡測試

1. 在已新增機器人的群組中
2. @機器人 傳送訊息
3. 等待機器人回覆

### 取得群組 Chat ID

如需取得群組 Chat ID 用於測試：

1. 打開群組設定
2. 查看「群資訊」
3. 複製群號（以 `oc_` 開頭的字串）

## ❓ 常見問題

### Q: 長連線儲存失敗？

**A**: 確保 OpenClaw 服務已啟動。飛書後台需要偵測到客戶端連線才能儲存長連線設定。

```bash
# 啟動服務
openclaw gateway start

# 查看狀態
openclaw gateway status
```

### Q: 機器人不回覆訊息？

檢查以下幾點：

1. **服務是否運行**: `openclaw gateway status`
2. **事件訂閱是否配置**: 確保新增了 `im.message.receive_v1` 事件
3. **權限是否完整**: 確保新增了 `im:message` 和 `im:message:send_as_bot` 權限
4. **應用是否發布**: 未發布的應用無法正常使用
5. **機器人是否在群裡**: 確保機器人已新增到群組

### Q: 群聊中如何觸發機器人？

預設情況下，群聊中需要 **@機器人** 才會觸發回覆，這樣可以避免機器人回應所有訊息。

### Q: 私聊機器人不回覆？

1. 確保你在應用的「可用範圍」內
2. 嘗試在飛書中搜尋機器人名稱，點擊進入私聊
3. 傳送訊息測試

### Q: 如何查看日誌排查問題？

```bash
# 查看即時日誌
openclaw logs --follow

# 執行診斷
openclaw doctor
```

## 🔧 進階配置

### 修改群組回應策略

預設群聊需要 @機器人 才回應。如需修改：

```bash
# 設定不需要 @機器人也回應（謹慎使用）
openclaw config set channels.feishu.requireMention false
```

### 切換到國際版 Lark

如使用國際版 Lark 而非飛書：

```bash
openclaw config set channels.feishu.domain "lark"
```
