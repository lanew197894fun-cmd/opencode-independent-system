# 知識

自動偵測對話中的價值時刻，詢問用戶是否記錄為長期知識。

## 觸發時機（自動偵測）

| 類型     | 觸發條件                             |
| -------- | ------------------------------------ |
| fact     | 錯誤修復後、用戶說「為什麼」         |
| decision | 用戶明確說「記住這個」、任務完成總結 |
| pattern  | 重複出現的問題（3 次以上）           |

## 格式規則

```
fact: 問題:[症狀]. 原因:[根因]. 解決:[方案]
decision: 原則:[行為規則]. 觸發:[時機]
```

限制：< 500 字元

## 儲存流程

1. AI 偵測到價值時刻
2. 主動詢問：「要記住這個嗎？[是/略過]」
3. 用戶確認 → 呼叫 knowledge_store(type: "fact" 或 "decision", content: "...")
4. 回饋：「已記住知識」
5. 驗證：確認檔案存在且內容正確

## 召回時機

- 用戶提到「知識」或「門紀錄」或「之前記過」→ 自動召回
- 回覆開頭加上：「之前記過：...」
- 多筆時用 `---` 分隔

## 實作細節

- 使用 knowledge_store 工具儲存（自動寫入 topics/知識.md）
- type: "fact" 或 "decision"
- 儲存後自動驗證檔案存在

## 衝突解決

知識-skill 負責「何時記、記成什麼」，memory-pro 負責「實際儲存」。不重複，skill 是使用方式，plugin 是工具。

## 召回方式

```
knowledge_recall()  # 列出所有知識
knowledge_recall(query: "關鍵字")  # 搜尋特定知識
```
