# [工具] Ubuntu 24.04 上可用的文字編輯器

## 系統環境

- 作業系統：Ubuntu 24.04.4 LTS
- 硬體：Acer Aspire E5-573G, Intel i5-5200U, 16GB RAM
- 韌體版本：V1.25

## 圖形介面文字編輯器 (GUI)

### 預設編輯器

- **gedit** - GNOME 預設文字編輯器
  - 特色：簡單易用，支援語法高亮，外掛擴充
  - 安裝：通常預裝，可 via `sudo apt install gedit`
  - 使用：`gedit filename.txt`

### 推薦編輯器

- **VS Code** - Microsoft Visual Studio Code
  - 特色：強大的程式編輯器，豐富外掛，整合終端機
  - 安裝：`sudo snap install --classic code` 或從官網下載 .deb
  - 使用：`code filename.txt`

- **Sublime Text** - 專業程式編輯器
  - 特色：快速響應，多選編輯，強大套件系統
  - 安裝：透過官網 APT repository
  - 使用：`subl filename.txt`

- **Atom** - GitHub 的可 hack 編輯器
  - 特色：高度客製化，套件生態豐富
  - 安裝：`sudo snap install --classic atom`
  - 使用：`atom filename.txt`

### 輕量級選項

- **Leafpad** - 極簡GTK+ 編輯器
- **Mousepad** - Xfce 預設編輯器
- **Geany** - 輕量級 IDE 特色編輯器

## 命令列介面編輯器 (CLI)

### 預設編輯器

- **nano** - 簡單易用的 CLI 編輯器
  - 特色：模式直觀，常用快捷鍵在畫面底部顯示
  - 安裝：通常預裝
  - 使用：`nano filename.txt`
  - 常用快捷鍵：
    - Ctrl+O: 儲存
    - Ctrl+X: 離開
    - Ctrl+W: 搜尋
    - Ctrl+K: 剪切行

### 高級編輯器

- **vim / neovim** - 強大的模態編輯器
  - 特色：極其強大，可擴充性高，隨處可用
  - 安裝：`sudo apt install vim neovim`
  - 使用：`vim filename.txt` 或 `nvim filename.txt`
  - 學習資源：`vimtutor` (內建教學)

- **emacs** - 可擴充的自描述編輯器
  - 特色：極其可客製化，幾乎可以做任何事
  - 安裝：`sudo apt install emacs`
  - 使用：`emacs filename.txt`

### 其他選項

- **micro** - 現代化直觀 CLI 編輯器
- **ed** - 標準 Unix 行編輯器 (腳本用)
- **red** - 改良版 ed

## 如何選擇編輯器

### 根據使用情境

1. **快速編輯設定檔**：使用 nano 或 vim
2. **程式開發**：使用 VS Code、Sublime Text 或 neovim
3. **筆記寫作**：使用 gedit、Atom 或典雅編輯器
4. **遠端伺服器**：必須使用 CLI 編輯器 (vim/nano/emacs)
5. **批次處理腳本**：使用 sed、ed 或 vim 的 Ex 模式

### 安裝建議

對於大多數用戶，推薦安裝：

```bash
# 基本編輯器 (通常已安裝)
sudo apt install vim nano

# 現代程式編輯器
sudo snap install --classic code  # VS Code
# 或
sudo snap install --classic sublime-text  # Sublime Text (需要授權)

# 或選擇開源方案
sudo snap install --classic atom
```

## 與 opencode 的整合

opencode 內建支援多種編輯器透過環境變數：

- `$EDITOR` - 預設編輯器 (通常設為 vim 或 nano)
- `$VISUAL` - 視覺編輯器 (優先級較高)

可透過以下方式設定：

```bash
# 暫時設定 for 本次 session
export EDITOR=code
export VISUAL=code

# 永久設定 (加入 ~/.bashrc 或 ~/.zshrc)
echo 'export EDITOR=code' >> ~/.bashrc
echo 'export VISUAL=code' >> ~/.bashrc
```

然後在 opencode 中使用編輯功能時會啟用指定的編輯器。

## 維護與更新

定期更新編輯器以獲得最新功能和安全修復：

```bash
# APT 包
sudo apt update && sudo apt upgrade vim nano gedit

# Snap 包
sudo snap refresh code  # VS Code
sudo snap refresh atom  # Atom
```

## 問題排除

### 常見問題

1. **中文顯示亂碼**：確保系統 locale 設定正確 (UTF-8)

   ```bash
   locale  # 檢查設定
   sudo dpkg-reconfigure locales  # 重新設定
   ```

2. **編輯器無法開啟檔案**：檢查檔案權限

   ```bash
   ls -l filename.txt
   chmod 644 filename.txt  # 若需要修改權限
   ```

3. **效能問題**：在舊硬體上使用輕量編輯器
   - 輕量選項：nano, vim, leafpad
   - 避免：某些 Electron 編輯器在低記憶體系統上可能吃緊
