## 功能說明

執行重要任務前自動檢測系統狀態，判斷是否符合執行條件。

---

## 檢測項目

| 項目     | 安全 | 警告   | 危險   | 臨界 |
| -------- | ---- | ------ | ------ | ---- |
| 記憶體   | <60% | 60-80% | 80-90% | >90% |
| CPU      | <50% | 50-80% | 80-95% | >95% |
| 磁碟空間 | >20% | 10-20% | 5-10%  | <5%  |

---

## 執行策略

| 狀態     | 策略               |
| -------- | ------------------ |
| 全部安全 | 完整並發執行       |
| 部分警告 | 適度並發，監控資源 |
| 有危險項 | 單線程執行         |
| 有臨界項 | 先改善再執行       |

---

## 快取優化

### 快取策略

| 項目     | TTL  | 說明          |
| -------- | ---- | ------------- |
| 系統狀態 | 30s  | 快速變動      |
| 資源報告 | 60s  | 1分鐘內不重複 |
| 優化結果 | 300s | 5分鐘內不重複 |

### 使用流程

```
執行前：
→ tool_cache_get(key: "system:status")
→ 有快取：使用快取結果
→ 無快取：執行檢測 + 快取結果
```

---

## 優化動作

### 記憶體

```bash
rm -rf node_modules/.cache
rm -rf /tmp/opencode-*
```

### 磁碟

```bash
find . -name "*.log" -mtime +7 -delete
rm -rf .opencode/tmp/*
```

### CPU

- 降低並發數量
- 延遲非必要任務

---

## 使用方式

- 自動：執行大型任務前檢測
- 手動：「檢查系統狀態」「系統健康檢查」

---

## 檢測指令

```bash
free -h        # 記憶體
top -bn1 | head -5  # CPU
df -h .        # 磁碟
uptime        # 系統負載
```
