# 通用除錯

## 核心原則

**任何問題**描述時，立即主動檢查，不要索要錯誤訊息。

## 通用診斷流程

### 1. 環境檢查

- 專案結構是否完整
- 依賴是否安裝（node_modules, bun.lock）
- 設定檔是否存在

### 2. 語法檢查

- 嘗試 lint/typecheck
- 檢查最近修改的檔案

### 3. 執行檢查

- 嘗試直接執行相關命令
- 檢查命令路徑

### 4. 假設驗證

- 先猜測可能原因
- 用工具驗證

## 禁止事項

- 不要問「請提供錯誤訊息」
- 不要假設使用者無法取得資訊
- **先檢查，再詢問**
