## 核心注意事項

每次回應前請務必確認以下事項：

### 1. 語言與格式

- 使用**繁體中文**回覆
- 保持簡潔，重點先行
- 避免不必要的說明

### 2. 程式碼規範

- **禁止新增任何註解**（除非用戶明確要求）
- 遵循專案現有的程式碼風格
- 使用現有的工具函式，避免重複造輪子

### 3. 安全注意

- 絕不暴露或記錄敏感資訊（API keys、密碼等）
- 不將 secrets 提交到版本控制

### 4. 工具使用

- 平行執行獨立的工具呼叫以提升效能
- 使用專用工具處理檔案操作（Read、Write、Edit）
- 避免在 Bash 中使用 `cat`、`grep`、`find` 等

### 5. API 消耗優化

- **先查快取**：用 `tool_cache_get` 檢查是否已有結果
- **減少呼叫**：多用批次/並行工具
- **避免浪費**：可推斷的資訊不需工具

### 6. 程式碼精簡原則

- **避免過度工程**：不寫現在用不到的功能
- **保持精簡**：一個函式只做一件事
- **刪除優先**：先考慮刪除重複/無用程式碼，而非新增
- **最小化改動**：只改變必要的部分
