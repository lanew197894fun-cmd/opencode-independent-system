# 獨立架構 (Independent Architecture)

**完全自主研發的私有化 AI 控制中心**

---

## 願景

建立一個**完全獨立、完全可控**的 AI 系統，不依赖任何外部服務商的閉源系統。

---

## 核心理念

| 理念           | 說明                          |
| -------------- | ----------------------------- |
| **自主研發**   | 100% 自己開發，不拿別人的產品 |
| **完全可控**   | 所有程式碼自有，安全可靠      |
| **插件化設計** | 靈活擴展，熱拔差              |
| **私有化部署** | 資料留在本地，不上雲          |

## 核心模組

### 1. Gateway - 網關

**職責**: 消息路由、頻道管理、訊息監聽

```
頻道 ──→ 網關 ──→ 路由規則 ──→ 處理器
         │
         └──→ 訊息歷史
```

### 2. Commander - 指揮官

**職責**: 服務管理、健康檢查、故障恢復

```
服務狀態 ──→ 指揮官 ──→ 健康檢查 ──→ 自動重啟
            │
            └──→ Web UI / CLI
```

### 3. SecurityGuard - 安全守衛

**職責**: 插件檢測、技能檢測、衝突檢測

```
插件/技能 ──→ 安全守衛 ──→ 掃描 ──→ 阻擋/放行
              │
              └──→ 通知用戶
```

### 4. UnifiedSecurityCenter - 安全中心

**職責**: VPN、RDP、24小時防禦、防詐騙

```
┌─────────────────────────────────────┐
│           安全中心                    │
├─────────────────────────────────────┤
│  🔒 VPN 管理      ───→ WireGuard    │
│  🖥️ RDP 防護     ───→ 3389 阻擋   │
│  ⏰ 24小時防禦   ───→ fail2ban     │
│  🚨 防詐騙檢測   ───→ 訊息分析     │
└─────────────────────────────────────┘
```

### 5. PluginSystem - 插件系統

**職責**: 插件管理、Hook 系統、熱拔差

```
插件 ──→ 註冊 ──→ Hook 管理 ──→ 執行
         │
         └──→ 安全檢查
```

## 插件架構

### 內建插件

| 插件      | 說明                                         |
| --------- | -------------------------------------------- |
| Channel   | 渠道整合 (Telegram, Discord, Slack, Webhook) |
| Memory    | 記憶系統 (短期/長期/偏好)                    |
| Knowledge | 知識庫 (存取/搜尋/經驗)                      |
| Tools     | 工具集 (檔案/命令/API)                       |
| Security  | 安全插件 (VPN/RDP/防禦)                      |

### 插件結構

```typescript
interface Plugin {
  metadata: {
    name: string
    version: string
    author: string
    description?: string
  }
  hooks: {
    [hookName: string]: HookFunction
  }
  tools?: ToolDefinition[]
  init?: () => Promise<void>
  shutdown?: () => Promise<void>
}
```

### Hook 列表

| Hook             | 觸發時機   |
| ---------------- | ---------- |
| `cli.init`       | CLI 初始化 |
| `cli.start`      | CLI 啟動   |
| `cli.shutdown`   | CLI 關閉   |
| `chat.before`    | 聊天前     |
| `chat.after`     | 聊天後     |
| `chat.error`     | 聊天錯誤   |
| `tool.before`    | 工具執行前 |
| `tool.after`     | 工具執行後 |
| `session.create` | 會話創建   |
| `session.save`   | 會話保存   |
| `error.handle`   | 錯誤處理   |

## 安全架構

### 多層防護

```
┌─────────────────────────────────────────────────────────────┐
│                        使用者輸入                            │
└──────────────────────────┬──────────────────────────────────┘
                           │
┌──────────────────────────▼──────────────────────────────────┐
│                    第一層: 內容安全                          │
│  • 特殊字元過濾    • 指令注入防護    • XSS 防護           │
└──────────────────────────┬──────────────────────────────────┘
                           │
┌──────────────────────────▼──────────────────────────────────┐
│                   第二層: 插件/技能安全                      │
│  • 程式碼掃描      • 危險模式檢測    • 衝突檢測           │
│  • 詐騙模式識別    • 可疑行為攔截                          │
└──────────────────────────┬──────────────────────────────────┘
                           │
┌──────────────────────────▼──────────────────────────────────┐
│                    第三層: 系統安全                          │
│  • VPN 防護       • RDP 防護       • 24小時防禦           │
│  • fail2ban      • 防火牆規則    • 審計日誌             │
└─────────────────────────────────────────────────────────────┘
```

### 危險模式檢測

| 類型       | 模式                                 |
| ---------- | ------------------------------------ | ---------- | --- |
| 程式碼執行 | `eval()`, `exec()`, `child_process`  |
| 檔案操作   | `rm -rf`, `chmod 777`                |
| 網路       | `curl                                | sh`, `wget | sh` |
| 系統       | `sudo su`, `process.exit`            |
| 資料庫     | `DROP TABLE`, `DELETE FROM`          |
| 詐騙       | API Key 外洩, 假冒登入, 加密貨幣盜取 |

## 命令列舉

### CLI 指令

```bash
# 指揮官
ia commander status      # 服務狀態
ia commander start all  # 啟動全部
ia commander health     # 健康檢查
ia commander web start  # Web UI

# 網關
ia gateway connect telegram  # 連接頻道
ia gateway send telegram "Hello"
ia gateway status

# 安全
ia security status      # 安全狀態
ia security vpn connect  # VPN 連接
ia security scam check   # 防詐騙

# 插件
ia plugin list          # 插件列表
ia plugin enable <name> # 啟用
ia plugin security <name> # 安全檢查
```
