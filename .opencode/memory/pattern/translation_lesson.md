# 翻譯錯誤經驗

## 教訓

### SQL 結構翻車
- 日期: 早期翻譯
- 問題: 把 SQL 表名、列名翻譯了
- 結果: 資料庫壞掉
- 教訓: 單引號內的內容才能翻譯，單引號外是 SQL 語法不能碰

### 物品名稱翻車
- 日期: 2026-03-18
- 問題: `축복받은 스냅퍼의 마법저항 反지` 被翻成 `祝福`
- 原因: 字典有錯誤翻譯覆蓋
- 教訓: 檢查翻譯結果，確認物品名稱完整

## 防呆機制

翻譯前先檢查:
1. 不要翻 SQL 結構 (表名、列名、INSERT INTO、VALUES 等)
2. 檢查物品名稱是否被截斷
3. 用樣本測試再全量翻譯
