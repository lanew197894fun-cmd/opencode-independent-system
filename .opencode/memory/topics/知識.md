  - swarm/clawteam: 需要多代理協作時

## fact 2026-03-25
## 問題
技能自動載入缺乏即時視覺回饋，使用者發送訊息後系統看起來像當機，需等到 AI 回應才看到結果。

## 原因
- `skill-autoload.ts` 和 `intent-router.ts` 只將文字加入 `parts`，依賴 AI 回應後才顯示
- 沒有使用 TuiEvent.ToastShow 發布即時通知

## 修復
1. **intent-router.ts**：為「技能/skill」關鍵字檢測加入 toast
   - 載入中：variant: "info", duration: 3秒
   - 完成後：variant: "success", duration: 5秒

2. **skill-autoload.ts**：為首次訊息自動載入加入 toast
   - 初始化：variant: "info", duration: 3秒
   - 完成後：variant: "success", duration: 5秒

## 程式碼變更
```typescript
// skill-autoload.ts
import { TuiEvent } from "../cli/cmd/tui/event"
import { Bus } from "../bus"

// 載入前
Bus.publish(TuiEvent.ToastShow, {
  message: "正在初始化智慧技能系統...",
  variant: "info",
  duration: 3000,
})

// 完成後
Bus.publish(TuiEvent.ToastShow, {
  message: `已載入 ${loaded.length} 個技能`,
  variant: "success",
  duration: 5000,
})
```
